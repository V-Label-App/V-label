# Project Constitution: V-label App

This document defines the core principles, technology stack, and standards for the V-label application. All code generated by AI agents must adhere to this constitution.

## Core Principles

1.  **Strict Type Safety**: Use TypeScript strictly. No `any` types unless absolutely necessary. Interfaces for all data structures.
2.  **No Hardcoding**: All configuration must come from environment variables (`process.env`).
3.  **Environment Configuration**: Centralized config in `src/config/env.ts`.
4.  **Error Handling**: Graceful error handling with structured logging. No silent failures.
5.  **Database Security**: Always use parameterized queries to prevent SQL injection.
6.  **Code Organization**: Separation of concerns (Routes → Controllers/Services → Models → Utils).
7.  **Performance**: Use connection pooling, async/await properly.
8.  **Logging**: Use the standard `logger` utility, not `console.log`.

## Technology Stack

### Backend
-   **Runtime**: Node.js (Latest LTS)
-   **Framework**: Express.js
-   **Language**: TypeScript
-   **Database**: PostgreSQL 16 (Dockerized)
-   **ORM/Query**: Direct `pg` client with connection pooling (no heavy ORM preferred unless specified)
-   **Authentication**: JWT (JSON Web Tokens)
-   **Validation**: Zod environment validation

### Frontend
-   **Framework**: React 19
-   **Build Tool**: Vite
-   **Language**: TypeScript
-   **Styling**: Vanilla CSS (CSS Modules or standard CSS)
-   **State Management**: Zustand
-   **Canvas**: Konva / React-Konva

### Infrastructure
-   **Containerization**: Docker & Docker Compose
-   **OS**: macOS (Development environment)

## Standards & Conventions

### Naming
-   **Variables/Functions**: `camelCase` (e.g., `getUserById`)
-   **Constants**: `UPPER_SNAKE_CASE` (e.g., `MAX_RETRIES`)
-   **Types/Interfaces**: `PascalCase` (e.g., `UserProfile`)
-   **Files**: `kebab-case` (e.g., `user-controller.ts`)

### Code Structure
```
server/src/
├── config/          # Environment & App config
├── middlewares/     # Express middlewares
├── routes/          # API route definitions
├── services/        # Business logic
├── models/          # Data models & Types
├── utils/           # Helper functions
└── migrations/      # SQL migration files
```

### Database Migrations
-   Never modify existing migrations.
-   Create new migrations using `npm run migration:create`.
-   Run migrations using `npm run migration`.
